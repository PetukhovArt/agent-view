## Iteration 2: Interaction — COMPLETED (2026-03-29)

Commands implemented: `launch`, `click`, `fill`, `screenshot`

### What works
- `agent-view launch` — spawns app from config.launch, polls CDP until ready, 60s timeout
- `agent-view launch` — idempotent: if already running, prints status and exits
- `agent-view click <ref>` — click DOM element by session ref ID (scrolls into view, clicks center)
- `agent-view click --pos <x,y>` — click by coordinates (for canvas)
- `agent-view fill <ref> <value>` — set input value with native setter + dispatch input/change events
- `agent-view screenshot` — captures PNG via CDP, saves to temp dir, prints path
- All commands support `--window <id|name>` for multiwindow targeting
- Invalid refs return clear error message suggesting `dom` refresh

### Implementation details
- Window resolution refactored into reusable `resolveWindow()` + `getConnection()` in server
- Click uses `DOM.resolveNode` → `scrollIntoViewIfNeeded` → `DOM.getBoxModel` → center coordinates → `Input.dispatchMouseEvent`
- Fill uses native value setter (HTMLInputElement/HTMLTextAreaElement prototype) for framework reactivity
- Launcher spawns detached process with shell, polls listTargets every 1s

### Tested on web-client (Electron + Vue 3)
- discover ✓, screenshot ✓ (43KB PNG), click by ref ✓, click by pos ✓, launch idempotency ✓
- fill error handling ✓ (invalid ref returns message)
- launch from cold start: timeout on 60s — likely npm/electron-vite startup overhead, needs investigation

## Iteration 3: WebGL — COMPLETED (2026-03-29)

Commands implemented: `scene`, `snap`

### What works
- `agent-view scene` — reads PixiJS scene graph via `window.__PIXI_DEVTOOLS__` through CDP Runtime.evaluate
- `agent-view scene --filter <text>` — filter by object name or type
- `agent-view scene --depth <N>` — limit tree depth
- `agent-view scene --verbose` — extended properties (alpha, scale, rotation, bounds)
- `agent-view scene --diff` — returns only changes vs previous call (cached in server memory)
- `agent-view snap` — combined DOM + Scene output in one call, clearly separated sections
- Graceful fallback when no PixiJS detected: "No PixiJS scene found"

### Implementation details
- Scene graph extracted via JS evaluated in browser context, reads `__PIXI_DEVTOOLS__.app.stage`
- Serializes recursively: type, name/label, position, visible, tint, alpha, scale, rotation, bounds
- Diff compares by child key (type:name or type:index), reports added/removed/changed nodes
- Snap shows DOM section always, Scene section only when PixiJS is present

### Tested on web-client (Electron + Vue 3)
- scene returns clear message (no PixiJS in this app) ✓
- snap shows DOM-only when no PixiJS ✓
- PixiJS scene graph requires testing on SCADA project (Tauri + SolidJS + PixiJS)
